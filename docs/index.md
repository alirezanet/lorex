  - icon: 🔌
    title: Works with every agent
    details: Add a skill once. Lorex projects it into Claude, Copilot, Codex, Cursor, Cline, Windsurf, Roo, Gemini, and OpenCode — automatically, in each tool's native format.
  - icon: 🤝
    title: Shared team registry
    details: Connect any Git repo as your team's skill registry. Publish a skill once and every developer, every repo stays current with <code>lorex sync</code>.
  - icon: 🌐
    title: Community skill sources (Taps)
    details: Pull skills from any public Git repository with <code>lorex tap add</code>. No registry setup needed — connect a framework team's collection or an open-source library's conventions in one command.
  - icon: 🔒
    title: You stay in control
    details: No probabilistic retrieval, no black-box context injection. Skills are plain Markdown in your own Git repository. You decide exactly what your agents know and when it changes.
  - icon: 🌿
    title: Git-native versioning
    details: Skills are files in your repo. Review changes in PRs, diff history, roll back — everything you already know how to do with Git. Team registries support pull-request publish policies.
  - icon: 🤖
    title: Your AI can author skills too
    details: Lorex installs its own definition on <code>lorex init</code>, so your agent already knows the skill format. Ask it to analyse the codebase and write the first skill for you.
